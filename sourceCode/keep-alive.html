    # keep-alive

    # 1. 流程

    源码位置： vue\src\core\components\keep-alive.ts

    1. 初始化的方法

      • pruneCache：移除不匹配的组件
      • pruneCacheEntry：删除缓存的指定 Key 的组件
      • this.cacheVNode：缓存 Vnode 的组件实例

    2. 执行生命周期

      1. created：初始化数据
        • this.cache：对象，保存要缓存的组件对象
        • this.keys：数组，保存缓存的组件对象的的 key
      2. render：获取虚拟DOM
        1. 获取<keep-alive> 默认插槽传递的内容
        2. 检测组件是否需要缓存，是否在 include 中, 或在 exclude 中
          • 不缓存，返回 Vnode
          • 需要缓存
            • 在 this.cache 缓存过了，将缓存取出保存到 Vnode 中，执行 LRU 算法
            • 未缓存，先暂存到 this.vnodeToCache，this.keyToCache
      3. mounted：真正缓存组件的阶段，将暂存的
        • 调用 this.cacheVNode() 进行真正的缓存
          • 将暂存添加到真正的缓存 this.cache中
          • 将 标识符 key 添加到 this.keys 最后
          • 检测是否超出最大缓存数
            • 超出：使用 pruneCacheEntry 删除第一个缓存
            • 未超出：释放暂存的组件
        • 监听 include 和 exclude 这两个 Props，超出最大缓存数时，使用 pruneCache 移除不匹配项
      4. updated：调用 this.cacheVNode() 重新缓存当前的组件
      5. destroyed：调用 pruneCacheEntry 遍历销毁所有缓存的组件
